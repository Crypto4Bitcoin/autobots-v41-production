# Phase 13: Workflow Memory & Advice

This phase evolves AutoBots from a "smart generator" into a "smart advisor" by closing the loop between recommendation, user acceptance, and workflow outcome.

## User Review Required

> [!IMPORTANT]
> **Learning Loop**: Recommendations will now be weighted by historical success (`workflow_recommendation_feedback`) and user approval.
> **Persistence**: Assembled workflows will be stored as `workflow_proposals` to allow for asynchronous review and visual editing (Phase 14).

## Proposed Changes

### 1. Persistent Storage (Data Layer)
#### [NEW] `workflow_proposals` table
- Tracks draft workflows generated by the assembler.
- Fields: `id`, `workspace_id`, `proposed_by`, `goal_prompt`, `recommended_pack`, `recommended_template`, `draft_definition` (JSONB), `status` (pending/approved/rejected), `created_at`.

#### [NEW] `workflow_recommendation_feedback` table
- Tracks the effectiveness of the recommendation engine.
- Fields: `id`, `goal_text`, `template_id`, `pack_slug`, `outcome` (accepted/rejected/abandoned), `success_score` (from later execution), `created_at`.

### 2. Intelligent Services (Memory Layer)
#### [NEW] [workflow-memory-service.ts](file:///c:/Users/owner/.gemini/antigravity/scratch/src/lib/services/workflow-memory-service.ts)
- Analyzes historical outcomes to identify high-performing workflow patterns.
- Identifies "Success Profiles" for packs by workspace.

#### [NEW] [recommendation-feedback-service.ts](file:///c:/Users/owner/.gemini/antigravity/scratch/src/lib/services/recommendation-feedback-service.ts)
- Records operator interactions with recommendations and proposals.
- Periodically updates ranking weights for `WorkflowRecommendationService`.

### 3. Core Updates
#### [MODIFY] [workflow-recommendation-service.ts](file:///c:/Users/owner/.gemini/antigravity/scratch/src/lib/services/workflow-recommendation-service.ts)
- Incorporates historical ranking data into the confidence score calculation.
